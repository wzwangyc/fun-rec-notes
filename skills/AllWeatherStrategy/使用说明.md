# 🛡️ 全天候 ETF 资产配置系统 - 使用说明

## 📝 快速开始

### 1. 修改配置

打开 `config.ini` 文件，修改以下参数：

```ini
# ETF 配置池 - 每行一个 ETF 代码
ETF_LIST = [
    515450,  # 沪深 300ETF
    159545,  # 中证 500ETF
    513100,  # 纳斯达克 100ETF
    518880,  # 黄金 ETF
]

# 配置总金额（元）
TOTAL_AMOUNT = 10000

# 回看天数（用于计算历史波动率）
LOOKBACK_DAYS = 365

# 是否自动发送邮件
SEND_EMAIL = True

# 邮件接收地址
EMAIL_TO = "wangreits@163.com"
```

### 2. 运行策略

```bash
cd C:\Users\28916\.openclaw\workspace\skills\AllWeatherStrategy
py run_configurable.py
```

### 3. 查看结果

- **PDF 报告**：`全天候配置报告/AllWeather_Report_YYYYMMDD_HHMMSS.pdf`
- **邮件通知**：自动发送到配置的邮箱

## 🔧 高级配置

### 修改风险偏好

编辑 `scripts/engine.py` 中的 `RiskParityStrategy` 类：

```python
# 风险目标调整（默认 1.0 表示等风险贡献）
RISK_TARGET = 1.0  # >1.0 更激进，<1.0 更保守
```

### 自定义输出目录

编辑 `engine.py` 中的 `Config` 类：

```python
OUTPUT_DIR = "我的配置报告"  # 修改输出目录名称
```

## 🐛 常见问题

### Q1: 提示"ETF 代码不存在"
**解决：** 检查 ETF 代码是否正确，确保是 6 位数字

### Q2: 邮件发送失败
**解决：** 
1. 检查 SMTP 授权码是否正确
2. 确认邮箱已开启 SMTP 服务
3. 检查网络连接

### Q3: 计算结果不理想
**解决：**
1. 增加回看天数（如改为 730 天）
2. 调整 ETF 池，增加资产类别
3. 检查 ETF 是否有足够的历史数据

## 📈 策略原理

**全天候策略 (All Weather Strategy)** 基于 **Risk Parity (风险平配)** 原理：

1. **风险贡献均衡** - 每个资产对组合的风险贡献相等
2. **穿越牛熊** - 通过分散配置应对不同市场环境
3. **长期稳健** - 不追求最高收益，追求风险调整后最优收益

### 资产配置逻辑

```
股票类 (沪深 300 + 中证 500 + 纳指) - 进攻型资产
债券类 (国债 ETF) - 防守型资产
商品类 (黄金 + 豆粕) - 抗通胀资产
```
