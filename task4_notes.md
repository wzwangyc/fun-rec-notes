## 学习笔记：生成式推荐的 Tokenizer 技术
### 1. 物品表示范式的演进
推荐系统正在经历从“死记硬背”到“逻辑理解”的范式转移。

稀疏 ID (Atomic ID): 给每个物品发一个“身份证号”。

痛点: 词表随物品数量线性爆炸；新物品（冷启动）没有历史数据，模型完全不认识。

文本 ID (Text-Based): 用自然语言描述物品。

痛点: 序列太长，Transformer 计算量平方级增加；生成的内容可能在现实中不存在。

语义 ID (Semantic ID): 主流选择。将物品化整为零，编码成一串有意义的数字序列。

### 2. 核心离散化技术：从 VQ 到 RQ
如何把高维的连续向量（Embedding）变成离散的 Token 序列？

VQ-VAE (基础款): 准备一个码本（Codebook），把连续向量“强行”对应到最近的离散码字上。

核心: 通过直通估计器 (STE) 解决了离散化不可导的训练难题。

RQ-VAE (进阶款): 引入“残差”概念。第一层量化主要语义，第二层量化第一层剩下的“误差”，以此类推。

优势: 用极小的码本实现指数级的表达能力。它像一个不断对焦的过程：

第一层：看清是大类（体育）

第二层：看清是子类（篮球）

第三层：看清是细节（扣篮）

### 3. 工业级优化：RQ-Kmeans 与 RQ-OPQ
在大规模生产环境下，纯端到端的模型太重，需要更灵活的方案。

RQ-Kmeans: 将“学习码本”改为“聚类码本”。先用 BERT/CLIP 提特征，再做 K-means 聚类。

工程意义: 算法解耦，新物品上线只需做向量检索即可生成 ID，无需重训模型。

RQ-OPQ (混合编码): 针对搜索场景的杀手锏。

分工: RQ 负责纵向的层级（共性），OPQ 负责横向的属性（个性，如：颜色、内存）。

效果: 既能保证相似物品聚在一起，又能区分开具体的不同规格（SKU）。
